# AGENTS.md — `internal/infrastructure/postgres/` (persistence adapter)

PostgreSQL implementations of the `boundaries/repository` ports, built on
**pgx/v5** and **sqlc**-generated code. This is the outer ring — it may import
anything, but nothing inward may import it.

## sqlc — generated, do not hand-edit

Config: `/sqlc.yaml` (engine `postgresql`, driver `pgx/v5`, `emit_interface`,
`emit_empty_slices`, `emit_db_tags`).

- **Schema source = the migrations** (`sqlc/migrations/`). Migrations are the
  schema of record; sqlc reads them to type queries.
- **Queries you write:** `sqlc/queries/sql/*.sql`, annotated `-- name: X :one|:many|:exec`.
- **Generated output:** `sqlc/queries/*.sql.go`, `models.go`, `querier.go`,
  `db.go` — all carry `// Code generated by sqlc. DO NOT EDIT.`

Workflow: edit the `.sql`, run **`make sqlc`**, commit both the `.sql` and the
regenerated `.sql.go`. Never edit the generated Go by hand.

## Repository implementations

Each repo wraps a `*queries.Queries` (over a `pgxpool.Pool`) and converts between
domain types and sqlc rows:

```go
type RideRepo struct { q *queries.Queries; graceMins int32 }
func NewRideRepo(pool *pgxpool.Pool, graceMins int) *RideRepo { ... }
```

- `NewPool()` (`db.go`) reads `DATABASE_URL` and pings.
- `RideRepo`/`RequestRepo` take a `graceMins` arg (how long past a ride's flex
  window it stays listed); wired from `RIDE_GRACE_MINUTES` in `main.go`.
- **All domain⇄pgtype conversion goes through `convert.go`** (`uuidFrom/uuidTo`,
  `tsFrom/tsTo`, `dateFrom/dateTo`, `rideFromRow`, …). pgx nullables
  (`pgtype.UUID/Timestamptz/Date`) map to Go zero values when `!Valid` — a zero
  `time.Time` is the domain's "NULL". Add new conversions here, not inline.

## Migrations

`sqlc/migrations/NNN_name.up.sql` + matching `NNN_name.down.sql`, embedded via
`migrations.go` (`go:embed`), applied by `cmd/migratedb` (golang-migrate).

- Sequential 3-digit prefix. **Every `up` needs a `down`.** Both files, always.
- Run with `go run ./cmd/migratedb up` (also the boot command in `Procfile`).
- Numbering note: `main` skips 007 (006 → 008) — `007` lives on a feature branch,
  not a mistake to "fix" by renumbering. golang-migrate tracks applied versions by
  name and tolerates the gap. **Do not reuse 007**; the next migration is the
  highest existing number + 1.

## vapid/ & webpush/ (siblings, same outer ring)

- `vapid/resolve.go` — `Resolve()` returns the Web Push VAPID keypair, preferring
  DB-stored keys, then env (`VAPID_*`), else generating and persisting a fresh
  pair race-safely (so subscribers stay valid across restarts). Backed by
  `postgres.SettingsRepo` via a small `Store` interface.
- `webpush/webpush.go` — `Notifier` implementation; encrypts and POSTs payloads
  to subscription endpoints. A `410 Gone` means the subscription is dead (callers
  prune it).

## Integration tests

`*_integration_test.go` here are guarded by `//go:build integration` and need a
real Postgres (`TEST_DATABASE_URL`, falling back to `DATABASE_URL`). Run via
`make test` (which adds `-tags integration -p 1` so tests share the DB serially).

- `integration_test.go` holds the shared `TestMain` (opens `testPool`, waits until
  the schema is migrated) and a `truncate(t)` helper that clears all tables.
- Each test starts with `truncate(t)`, then exercises a real `NewXxxRepo(testPool, …)`.
